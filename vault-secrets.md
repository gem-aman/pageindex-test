# Vault Secrets Management

This document describes how HashiCorp Vault is used to manage secrets across our platform. Vault serves as the single source of truth for all sensitive credentials.

## Vault Architecture

### Server Setup

- Vault runs in HA mode with 3 nodes
- Backend: Consul storage
- Auto-unseal: Azure Key Vault (see [Azure Infrastructure Guide](azure-infrastructure.md#azure-key-vault) for the Key Vault setup)
- Audit logging: Enabled (file + syslog)

### Access Methods

| Method | Use Case | Reference |
|--------|----------|-----------|
| AppRole | Jenkins CI/CD | [Jenkins CI/CD Pipeline](jenkins-pipeline.md#vault-authentication) |
| Azure Auth | AKS workloads | [Azure Infrastructure Guide](azure-infrastructure.md#secrets-integration) |
| OIDC | Developer access | Internal SSO |
| Token | Terraform automation | [Terraform Modules Guide](terraform-modules.md#prerequisites) |

## Secret Engines

### KV v2 Engine

We use the KV v2 secrets engine with the following path structure:

```
secret/
├── platform/
│   ├── dev/
│   │   ├── azure-sql/        # SQL credentials
│   │   ├── acr/              # Container registry credentials
│   │   ├── kubeconfig/       # AKS kubeconfig
│   │   └── app-secrets/      # Application-specific secrets
│   ├── staging/
│   │   └── ... (same structure)
│   └── prod/
│       └── ... (same structure)
├── terraform/
│   ├── service-principal/    # Azure SP for Terraform
│   └── state-backend/        # Storage account keys
└── jenkins/
    ├── github-token/         # GitHub PAT for repo access
    └── docker-hub/           # Docker Hub credentials
```

### Database Secrets Engine

For dynamic database credentials:

```hcl
vault write database/config/azure-sql \
  plugin_name=mssql-database-plugin \
  connection_url="sqlserver://{{username}}:{{password}}@sql-platform-prod.database.windows.net:1433" \
  allowed_roles="flyway-migration,app-readonly"
```

Dynamic credentials are used by Flyway for migrations — see [Flyway Database Migrations](flyway-migrations.md#vault-dynamic-credentials).

## Specific Secret Paths

### Azure SQL Credentials

Path: `secret/platform/{env}/azure-sql`

```json
{
  "username": "sqladmin",
  "password": "<generated>",
  "server": "sql-platform-{env}.database.windows.net",
  "database": "app_db"
}
```

Used by:
- Terraform SQL module — [Terraform Modules Guide](terraform-modules.md#sql-database-module)
- Flyway migrations — [Flyway Database Migrations](flyway-migrations.md#database-connection)
- Application runtime (via dynamic credentials)

### ACR Credentials

Path: `secret/platform/{env}/acr`

```json
{
  "login_server": "acrplatform{env}.azurecr.io",
  "username": "<managed-identity>",
  "token": "<generated>"
}
```

Used by:
- Jenkins Docker build — [Jenkins CI/CD Pipeline](jenkins-pipeline.md#docker-build-and-push)
- Terraform ACR module — [Terraform Modules Guide](terraform-modules.md#acr-module)

### Kubeconfig Storage

Path: `secret/platform/{env}/kubeconfig`

Contains the AKS cluster kubeconfig for each environment. Generated by Terraform and stored after `terraform apply`.

Used by:
- Jenkins for Kubernetes deployments — [Jenkins CI/CD Pipeline](jenkins-pipeline.md#kubernetes-deployment)
- Flyway for running migrations in-cluster — [Flyway Database Migrations](flyway-migrations.md#running-migrations-in-kubernetes)

### Terraform Service Principal Credentials

Path: `secret/terraform/service-principal`


Used by:
- Terraform Azure provider — [Terraform Modules Guide](terraform-modules.md#prerequisites)

## Vault-to-Azure KeyVault Sync

A scheduled process syncs selected secrets from HashiCorp Vault to Azure Key Vault, enabling AKS pods to access secrets via the CSI Secrets Store Driver.

### Sync Process

1. Jenkins job runs every 15 minutes — see [Jenkins CI/CD Pipeline](jenkins-pipeline.md#vault-sync-job)
2. Reads secrets from Vault KV v2 engine
3. Writes to Azure Key Vault via Azure SDK
4. Verifies sync integrity with checksums

### Synced Secrets

| Vault Path | Azure KV Secret Name | Consumer |
|------------|---------------------|----------|
| `secret/platform/{env}/app-secrets/db-connection` | `db-connection-string` | AKS pods |
| `secret/platform/{env}/app-secrets/api-keys` | `external-api-keys` | AKS pods |
| `secret/platform/{env}/app-secrets/encryption-key` | `data-encryption-key` | AKS pods |

The Azure Key Vault is provisioned by Terraform — [Terraform Modules Guide](terraform-modules.md#key-vault-module).

## Vault Policies

### Jenkins Policy

```hcl
path "secret/data/platform/*" {
  capabilities = ["read", "list"]
}

path "secret/data/terraform/*" {
  capabilities = ["read", "list"]
}

path "secret/data/jenkins/*" {
  capabilities = ["read", "list"]
}

path "database/creds/flyway-migration" {
  capabilities = ["read"]
}
```

### Terraform Policy

```hcl
path "secret/data/platform/*/azure-sql" {
  capabilities = ["read"]
}

path "secret/data/terraform/*" {
  capabilities = ["read"]
}
```

### Flyway Policy

```hcl
path "database/creds/flyway-migration" {
  capabilities = ["read"]
}

path "secret/data/platform/*/azure-sql" {
  capabilities = ["read"]
}
```

See [Flyway Database Migrations](flyway-migrations.md#vault-dynamic-credentials) for how Flyway uses these credentials.

## Secret Rotation

### Rotation Schedules

| Secret | Rotation Period | Method |
|--------|----------------|--------|
| Azure SP credentials | 90 days | Vault auto-rotation |
| SQL admin password | 30 days | Manual via Jenkins job |
| ACR tokens | 7 days | Vault auto-rotation |
| Kubeconfig | On AKS changes | Terraform output |

### Rotation Procedure

1. Vault rotates the secret at the scheduled time
2. The Vault-to-Azure sync job propagates changes — see [Jenkins CI/CD Pipeline](jenkins-pipeline.md#vault-sync-job)
3. AKS pods pick up new secrets within 2 minutes (CSI driver polling)
4. Flyway migrations use dynamic credentials, so no rotation needed — see [Flyway Database Migrations](flyway-migrations.md#vault-dynamic-credentials)

## Troubleshooting

### Common Issues

1. **"Permission denied"**: Check Vault policy for the authentication method being used
2. **Sync lag**: The Jenkins sync job runs every 15 minutes — see [Jenkins CI/CD Pipeline](jenkins-pipeline.md#vault-sync-job)
3. **Expired credentials**: Check rotation schedule above; for Terraform SP, see [Terraform Modules Guide](terraform-modules.md#prerequisites)
4. **AKS pods can't read secrets**: Verify Key Vault access policies — see [Azure Infrastructure Guide](azure-infrastructure.md#azure-key-vault)
